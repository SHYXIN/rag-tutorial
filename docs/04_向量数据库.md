# 向量数据库

## 什么是向量数据库？

**专门存储和检索向量的数据库。**

```
传统数据库：存储文本、数字 → 精确匹配（WHERE name = '张三'）
向量数据库：存储向量 → 相似度搜索（找和这个向量最像的 10 个）
```

## 核心概念

### 1. 向量（Vector）

```
"沈令月是公主" → [0.12, -0.34, 0.56, ..., 0.78]  (512 维)
```

### 2. 索引（Index）

向量数据库用索引结构加速搜索，常见的有：

| 索引类型 | 原理 | 速度 | 精度 |
|----------|------|------|------|
| **HNSW** | 导航小世界图（类似跳表） | 快 | 高（近似最近邻） |
| **IVF** | 倒排文件索引（先聚类再搜索） | 很快 | 中（需要权衡 nprobe） |
| **Flat** | 暴力搜索（逐个比较） | 慢 | 精确 |
| **PQ** | 乘积量化（压缩向量） | 很快 | 中（有损压缩） |

实际生产中最常用的是 **HNSW**（兼顾速度和精度）。

### 3. ANN（近似最近邻）

不找"绝对最近"的（太慢了），找"差不多最近"的。

```
100 万个向量，逐个比较相似度 → O(n)，太慢
HNSW 索引 → O(log n)，快几百倍
```

## 常见向量数据库

| 数据库 | 类型 | 特点 |
|--------|------|------|
| **Chroma** | 本地/云 | 轻量级，Python 原生，适合本地开发 |
| **Milvus** | 开源分布式 | 高性能，适合大规模生产 |
| **Pinecone** | 云服务 | 全托管，无需运维 |
| **Qdrant** | 开源 | 支持过滤条件，性能优秀 |
| **pgvector** | PostgreSQL 插件 | 在现有 PG 上加向量能力 |
| **Weaviate** | 开源 | 支持多模态，GraphQL 查询 |

## Chroma 使用示例

```python
from langchain_chroma import Chroma
from embedding import LocalBGEEmbedding

embed_model = LocalBGEEmbedding(MODEL_DIR)

# 创建/加载数据库
db = Chroma(
    persist_directory="./chroma_db",
    embedding_function=embed_model,
    collection_name="my_docs"
)

# 添加文档
db.add_texts(
    texts=["沈令月是公主", "谢初是昭武将军"],
    metadatas=[{"source": "novel"}, {"source": "novel"}]
)

# 检索
results = db.similarity_search("沈令月的驸马是谁？", k=3)
for doc in results:
    print(doc.page_content)

# 带分数的检索
results = db.similarity_search_with_score("沈令月", k=3)
for doc, score in results:
    print(f"[{score:.3f}] {doc.page_content}")
```

## HNSW 索引原理（简化版）

```
构建阶段：
  1. 每个向量作为一个节点
  2. 建立多层图结构（类似跳表）
  3. 上层稀疏（快速定位区域），下层密集（精确搜索）

搜索阶段：
  1. 从顶层入口点开始
  2. 逐层向下，每层找最近的邻居
  3. 到达底层后，在局部区域精确搜索
  4. 返回 Top K 最近邻

复杂度：O(log n) vs 暴力搜索 O(n)
```

## 实际经验

- **小规模（< 10 万文档）**：Chroma + HNSW，完全够用
- **中规模（10 万 - 1000 万）**：Milvus 或 Qdrant
- **大规模（> 1000 万）**：Milvus 分布式 或 Pinecone
- **已有 PostgreSQL**：直接用 pgvector，不用额外维护

## 性能调优

```python
# Chroma 默认使用 HNSW，可以调参数
collection = client.create_collection(
    name="my_docs",
    metadata={
        "hnsw:space": "cosine",      # 相似度度量：cosine / l2 / ip
        "hnsw:construction_ef": 200,  # 构建时搜索深度（越大越精确，越慢）
        "hnsw:search_ef": 50,         # 搜索时搜索深度
        "hnsw:M": 16,                 # 每层最大连接数
    }
)
```
