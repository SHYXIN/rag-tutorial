# 混合检索（Hybrid Search）

## 问题背景

BM25 关键词检索和向量语义检索，各有各的盲区：

```
用户问："沈令月的驸马谢初的封号是什么？"

BM25 检索：
  ✅ 能找到包含"谢初"、"封号"、"驸马"的文档
  ❌ 如果文档里只有"谢初"没有"封号"这个词，就漏了

向量检索：
  ✅ 能理解"封号"≈"将军"≈"头衔"
  ❌ 可能返回描述谢初性格的文档（语义漂移）
```

## 核心思路

**BM25 + 向量，取并集，再 Rerank 精排。**

```
问题 → BM25 检索(Top K) ─┐
                          ├→ 合并去重 → Rerank → Top K
问题 → 向量检索(Top K) ──┘
```

## BM25 原理

BM25 是一种关键词检索算法，基于词频（TF）和逆文档频率（IDF）：

```
score(D, Q) = Σ IDF(qi) × (tf(qi, D) × (k1 + 1)) / (tf(qi, D) + k1 × (1 - b + b × |D| / avgdl))

其中：
  qi = 查询中的第 i 个词
  D = 文档
  tf = 词频（词在文档中出现的次数）
  IDF = 逆文档频率（词在所有文档中的稀有程度）
  k1, b = 调节参数（通常 k1=1.5, b=0.75）
  |D| = 文档长度
  avgdl = 平均文档长度
```

简单说：
- 词在文档中出现次数越多 → 分数越高
- 词在所有文档中越稀有 → 分数越高
- 文档越长 → 分数适当降低

## 实现

```python
class BM25Retriever:
    def __init__(self, documents):
        """构建 BM25 索引"""
        self.documents = documents
        self.k1, self.b = 1.5, 0.75

        # 对所有文档分词
        self.doc_tokens = []
        for doc in documents:
            tokens = self._tokenize(doc.page_content)
            self.doc_tokens.append(tokens)

        # 计算文档频率（DF）
        self.df = defaultdict(int)
        for tokens in self.doc_tokens:
            for token in set(tokens):
                self.df[token] += 1

        self.avg_dl = sum(len(t) for t in self.doc_tokens) / len(self.doc_tokens)
        self.N = len(self.doc_tokens)

    def _bm25_score(self, query_tokens, doc_idx):
        """计算单个文档的 BM25 分数"""
        tokens = self.doc_tokens[doc_idx]
        tf = defaultdict(int)
        for t in tokens:
            tf[t] += 1

        score = 0
        for qt in query_tokens:
            if qt not in tf:
                continue
            idf = math.log((self.N - self.df.get(qt, 0) + 0.5) / (self.df.get(qt, 0) + 0.5) + 1)
            tf_val = tf[qt]
            numerator = tf_val * (self.k1 + 1)
            denominator = tf_val + self.k1 * (1 - self.b + self.b * len(tokens) / self.avg_dl)
            score += idf * numerator / denominator
        return score

    def retrieve(self, query, k=5):
        query_tokens = self._tokenize(query)
        scores = [(i, self._bm25_score(query_tokens, i)) for i in range(self.N)]
        scores.sort(key=lambda x: x[1], reverse=True)
        return [self.documents[i] for i, s in scores[:k] if s > 0]


class HybridRetriever:
    def retrieve(self, query, k=5, bm25_weight=0.3):
        # BM25 检索
        bm25_results = self.bm25.retrieve(query, k=k)
        # 向量检索
        vector_results = self.db.similarity_search(query, k=k)
        # 合并去重 + 加权排序
        merged = self._merge(bm25_results, vector_results, bm25_weight)
        return merged[:k]
```

## 效果对比

```
问题："沈令月的驸马是谁？"

BM25 召回：
  ✅ 包含"驸马"、"沈令月"、"谢初"的文档
  ❌ "她嫁给了那个将军" → 匹配不上

向量召回：
  ✅ "她选择了谢初作为伴侣" → 语义相关
  ❌ 可能返回描述谢初性格的文档

混合召回 = BM25 ∪ 向量：
  ✅ 两种方式的结果都保留
  ✅ 覆盖面最广，不漏掉任何可能相关的文档
```

## 适用场景

- 知识库较大，单一检索方式容易漏掉
- 用户问题可能包含精确关键词（人名、术语、ID）
- 对召回率要求极高的场景

## 实际生产中的最佳实践

```
混合检索 → Rerank 精排 → 上下文压缩 → LLM 生成
   ↑              ↑              ↑            ↑
 覆盖面最广      精度最高       噪音最少      答案最好
```
