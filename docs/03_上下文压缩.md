# 上下文压缩（Context Compression）

## 问题背景

检索到的文档块里有很多无关内容：

```
检索到的文档（300 字符）：
"沈令月最近很烦恼，非常烦恼。她不明白，这长林盛宴本就是用来给她选夫择驸马用的...
（中间大段无关描述）...
那谢少将军不过是在长林盛宴中拔得了头筹，这位三公主就毫不犹豫地转头投入了新欢的怀抱..."

真正有用的句子只有 1-2 句，但 LLM 看到的是整段文字。
```

问题：
1. **Token 爆炸** — 多个文档块拼在一起，context window 装不下
2. **噪音干扰** — 无关句子会带偏 LLM
3. **成本增加** — 输入 token 越多，API 费用越高

## 核心思路

**从文档中提取与问题最相关的句子，只把关键句喂给 LLM。**

```
检索到的文档(长) → 句子分割 → 分数过滤(粗筛) → LLM精选(精筛) → 压缩后的上下文
```

## 两阶段压缩

### 阶段 1：分数过滤（快）

```python
def _score_filter(self, sentences, question):
    """计算每个句子与问题的相似度，保留 Top N"""
    q_emb = embed_model.embed_query(question)
    scored = []
    for sent in sentences:
        s_emb = embed_model.embed_query(sent)
        score = cosine_similarity(q_emb, s_emb)
        scored.append((sent, score))
    scored.sort(key=lambda x: x[1], reverse=True)
    return [s[0] for s in scored[:self.top_n_filter]]
```

### 阶段 2：LLM 精选（准）

```python
COMPRESS_PROMPT = """从以下候选句子中选出与问题最相关的 1-2 句。
只输出句子原文，不要解释。如果都不相关，输出"无相关信息"。"""

def _llm_select(self, candidates, question):
    cand_text = "\n".join(f"[{i+1}] {s}" for i, s in enumerate(candidates))
    return call_llm([
        {"role": "system", "content": COMPRESS_PROMPT},
        {"role": "user", "content": f"问题：{question}\n\n候选：\n{cand_text}"}
    ])
```

## 效果

```
压缩前：643 字符（3 个文档块，包含大量无关内容）
压缩后：323 字符（只保留关键句）
压缩比：50%

LLM 看到的上下文更短、更精准 → 答案质量更高
```

## 适用场景

- 检索到的文档块较长（> 200 字符）
- 多个文档块拼在一起 token 超限
- 对答案质量要求高，需要减少噪音

## 权衡

| 方案 | 速度 | 精度 | 成本 |
|------|------|------|------|
| 不压缩 | 快 | 低（噪音多） | 高（token 多） |
| 仅分数过滤 | 快 | 中 | 中 |
| 仅 LLM 压缩 | 慢 | 高 | 高（调多次 LLM） |
| 两阶段（推荐） | 中 | 高 | 中 |
