## 踩坑记录

### 坑 1：LongCat 没有 Embedding 模型

**现象：** 用 `OpenAIEmbeddings` 调 LongCat 的 `/v1/embeddings` 接口报错。

**原因：** LongCat 目前只提供 Chat 模型，没有 Embedding 模型。

**解决：** Embedding 改用本地 BGE 模型，LLM 生成用 LongCat Chat。

```python
# Embedding: 本地 BGE
from langchain_community.embeddings import HuggingFaceEmbeddings
embed_model = HuggingFaceEmbeddings(model_name="BAAI/bge-small-zh-v1.5")

# LLM: LongCat
from langchain_openai import ChatOpenAI
llm = ChatOpenAI(
    model="LongCat-2.0-Preview",
    base_url="https://api.longcat.chat/openai",  # 注意路径
    api_key="ak_xxx"
)
```

### 坑 2：HuggingFace 被墙，模型下载失败

**现象：** `sentence_transformers` 和 `fastembed` 都尝试从 HuggingFace Hub 下载模型，被代理挡住（SSL EOF 错误）。

**解决：** 从 Google Cloud Storage 手动下载 fastembed 打包的 ONNX 模型：

```bash
# 1. 下载（52MB）
curl -L -o model.tar.gz "https://storage.googleapis.com/qdrant-fastembed/fast-bge-small-zh-v1.5.tar.gz"

# 2. 解压到项目目录
mkdir -p models/bge-small-zh-v1.5
cd models/bge-small-zh-v1.5
tar xzf ../../model.tar.gz

# 目录结构：
# models/bge-small-zh-v1.5/fast-bge-small-zh-v1.5/
#   ├── model_optimized.onnx    (90MB)
#   ├── tokenizer.json
#   ├── config.json
#   └── vocab.txt
```

### 坑 3：Chroma 0.5+ 不再内置默认 Embedding

**现象：** `ValueError: You must provide an embedding function to compute embeddings`

**原因：** Chroma 0.5 之后移除了内置的 ONNX embedding，必须显式传入。

**解决：** 手写 `LocalBGEEmbedding` 类，用 ONNX Runtime 直接加载 BGE 模型：

```python
import onnxruntime as ort
from tokenizers import Tokenizer
import numpy as np

class LocalBGEEmbedding:
    def __init__(self, model_dir: str):
        self.tokenizer = Tokenizer.from_file(os.path.join(model_dir, "tokenizer.json"))
        self.session = ort.InferenceSession(
            os.path.join(model_dir, "model_optimized.onnx"),
            providers=["CPUExecutionProvider"]
        )

    def _embed(self, texts):
        encoded = self.tokenizer.encode_batch(texts)
        max_len = max(len(e.ids) for e in encoded)
        # padding...
        input_ids = np.zeros((len(encoded), max_len), dtype=np.int64)
        attention_mask = np.zeros((len(encoded), max_len), dtype=np.int64)
        token_type_ids = np.zeros((len(encoded), max_len), dtype=np.int64)
        for i, e in enumerate(encoded):
            input_ids[i, :len(e.ids)] = e.ids
            attention_mask[i, :len(e.attention_mask)] = e.attention_mask
            token_type_ids[i, :len(e.type_ids)] = e.type_ids

        outputs = self.session.run(None, {
            "input_ids": input_ids,
            "attention_mask": attention_mask,
            "token_type_ids": token_type_ids
        })

        # Mean pooling + L2 归一化
        last_hidden_state = outputs[0]
        mask_expanded = np.expand_dims(attention_mask, -1)
        sum_embeddings = np.sum(last_hidden_state * mask_expanded, axis=1)
        sum_mask = np.clip(np.sum(mask_expanded, axis=1), a_min=1e-9, a_max=None)
        embeddings = sum_embeddings / sum_mask
        norms = np.linalg.norm(embeddings, axis=1, keepdims=True)
        return embeddings / np.clip(norms, a_min=1e-9, a_max=None)
```

### 坑 4：Python httpx 走代理时 SSL 被干扰

**现象：** `curl` 调 LongCat API 正常，但 Python 的 `httpx`/`requests` 报 `SSL: EOF occurred in violation of protocol`。

**原因：** 系统代理对所有 HTTPS 出站流量做 MITM，Python 的 SSL 库与代理不兼容。

**方案 A（推荐）：** 用 `urllib.request` + 绕过 SSL 验证：

```python
import ssl
ctx = ssl.create_default_context()
ctx.check_hostname = False
ctx.verify_mode = ssl.CERT_NONE
resp = urllib.request.urlopen(req, context=ctx)
```

**方案 B：** 清除代理环境变量（不一定有效，取决于代理类型）：

```python
for k in ['HTTP_PROXY', 'HTTPS_PROXY', 'http_proxy', 'https_proxy']:
    os.environ.pop(k, None)
```

### 坑 5：Windows 终端编码问题

**现象：** `UnicodeEncodeError: 'gbk' codec can't encode character`

**原因：** Windows 终端默认 GBK 编码，无法输出 emoji 和特殊字符。

**解决：** 代码中去掉所有 emoji，用 ASCII 符号替代：

```python

```

## 完整运行流程

```bash
# 1. 创建虚拟环境
python -m venv .venv

# 2. 安装依赖
.venv/Scripts/pip install -r requirements.txt
.venv/Scripts/pip install sentence-transformers

# 3. 手动下载 BGE 模型（见坑2）

# 4. 配置 .env
echo OPENAI_API_KEY=ak_你的key > .env
echo OPENAI_BASE_URL=https://api.longcat.chat/openai >> .env

# 5. 导入文档
.venv/Scripts/python src/01_ingest.py

# 6. 问答
.venv/Scripts/python src/02_query.py
```
