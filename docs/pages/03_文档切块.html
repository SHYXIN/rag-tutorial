文档切块

切块策略、块大小选择、重叠的重要性

🎯 为什么需要切块？

LLM 的 context window 有限，不能把整篇文档塞进去。

一篇小说 = 400,000 字符
↓
LLM context window = 4,096 token ≈ 6,000 字符
↓
必须切成小块，每次只检索相关的小块

📏 切块策略

1. 固定长度切块（最简单）

from langchain.text_splitter import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=300,        # 每块 300 字符
    chunk_overlap=30,      # 块之间重叠 30 字符
    separators=["\n\n", "\n", "。", "！", "？", " ", ""]
)

2. 递归切块（推荐）

LangChain 的 RecursiveCharacterTextSplitter 按优先级尝试切分：

按 \n\n（段落）切分
↓ 太长
按 \n（换行）切分
↓ 太长
按 。（句子）切分
↓ 太长
按字符切分

📊 块大小的影响

块大小优点缺点
小（100-200）检索精准，噪音少可能丢失上下文，需要更多块
中（300-500）平衡精准度和上下文通用选择
大（800-1000）上下文完整可能包含无关内容，检索不够精准

🔗 重叠的重要性

无重叠

块1: "...沈令月选谢初为驸马。"
块2: "谢初在长林盛宴中..."
检索"谢初"时，块1 可能漏掉

有重叠（overlap=30）

块1: "...沈令月选谢初为驸马。谢初在..."
块2: "谢初在长林盛宴中拔得头筹..."
两个块都包含"谢初"，不会漏掉

💡 实际经验

小说/故事 — 按段落切分，chunk_size=300-500，overlap=30-50
技术文档 — 按章节/段落切分，chunk_size=500-800
FAQ — 每个问答作为一块（不需要再切）
代码 — 按函数/类切分
