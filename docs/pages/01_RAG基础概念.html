RAG 基础概念

什么是 RAG、完整流程、核心组件

🎯 什么是 RAG？

RAG（Retrieval-Augmented Generation，检索增强生成） = 检索系统 + 生成模型

核心思想：在让 LLM 回答问题之前，先从外部知识库中检索相关信息，然后把检索结果和问题一起提供给模型。

🧩 核心组件

组件作用常用选择
文档解析提取纯文本PyPDFLoader, TextLoader
文档切块切分成小块RecursiveCharacterTextSplitter
Embedding文本转向量BGE, OpenAI Embeddings
向量数据库存储和检索向量Chroma, Milvus, Pinecone
LLM生成回答GPT-4, Claude, LongCat

🆚 RAG vs 微调

维度微调RAG
知识存储模型权重内部外部知识库
知识更新需要重新训练更新知识库即可
答案可追溯❌ 无法追溯✅ 可追溯到文档
成本高（GPU 训练）低（检索 + API）
幻觉风险可能较低（有文档依据）

💡 经验法则

知识需要频繁更新 → 用 RAG
需要改变模型行为/风格 → 用微调
两者可以结合 → 先用微调优化模型，再用 RAG 提供知识
