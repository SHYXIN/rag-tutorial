Self-RAG（自我反思的检索增强生成）

让 LLM 自己决定何时检索、检索是否相关、回答是否有据

🎯 问题背景：普通 RAG 的三个痛点

传统 RAG 的流程是"无脑检索"——不管问题需不需要检索，都先去知识库搜一把。这会带来三个问题：

普通 RAG 的三大痛点

① 过度检索（Over-Retrieval）
用户问"你好"，系统也去检索 → 浪费资源，还可能引入无关信息干扰回答
② 检索不够（Under-Retrieval）
一次检索结果不够，但系统不知道 → 基于不完整信息回答，可能遗漏关键事实
③ 幻觉（Hallucination）
检索到的文档和问题无关，但 LLM 仍然基于无关文档生成回答 → 产生错误信息
痛点场景后果
过度检索闲聊、通用知识问题增加延迟和成本，引入噪音
检索不够复杂问题需要多角度信息回答不完整，遗漏关键信息
幻觉检索结果与问题无关LLM 基于无关文档编造答案
核心洞察：普通 RAG 的问题在于"检索"和"生成"之间没有反馈。Self-RAG 让 LLM 在每一步都能自我反思，决定下一步该怎么做。

💡 核心思路：LLM 自我反思

Self-RAG 的核心思想是：让 LLM 在生成过程中输出特殊的"反思 Token"，根据这些 Token 决定是否需要检索、检索结果是否相关、回答是否有文档支持。

Self-RAG vs 普通 RAG

普通 RAG：问题 → 检索 → 拼接 → 生成（一条路走到黑）
🆚
Self-RAG：问题 → 要不要检索？ → 检索 → 结果相关吗？ → 生成 → 回答有据吗？

🔑 三种反思 Token

Self-RAG 定义了三类反思 Token，让 LLM 在生成过程中输出：

反思类型Token作用触发时机
检索决策 [Retrieve] 需要检索 问题需要外部知识
[No Retrieve] 不需要检索 问题可直接回答
相关性判断 [Is Relevant] 检索结果与问题相关 检索完成后
[Not Relevant] 检索结果与问题无关 检索完成后
支持度判断 [Is Supported] 回答有文档支持 生成回答后
[Not Supported] 回答无文档支持 生成回答后

反思 Token 决策树

用户问题
↓ LLM 输出反思 Token
[Retrieve] → 执行检索 → 获取文档
[No Retrieve] → 直接生成回答
↓ 如果检索了，LLM 判断相关性
[Is Relevant] → 基于文档生成回答
[Not Relevant] → 重新检索 或 直接生成
↓ 生成回答后，LLM 判断支持度
[Is Supported] → 输出最终回答
[Not Supported] → 重新生成 或 标注"信息不足"

🔄 完整流程图

Self-RAG 完整执行流程

用户问题："谢初的官职是什么？"
↓ 步骤 1：检索决策
LLM 输出：[Retrieve]，因为问题涉及具体事实，需要检索
↓ 步骤 2：执行检索
检索结果 D1："谢初在长林盛宴中拔得头筹，被皇帝封为太傅。"
检索结果 D2："谢初出身寒门，凭借才华一步步晋升至太傅之位。"
↓ 步骤 3：相关性判断
D1 → [Is Relevant]（直接回答了官职）
D2 → [Is Relevant]（提到了太傅，但侧重晋升过程）
↓ 步骤 4：自适应生成
LLM 基于 D1 和 D2 生成回答："谢初官至太傅，他出身寒门，凭借才华一步步晋升。"
↓ 步骤 5：支持度判断
[Is Supported]："太傅"在 D1 中有明确支持，"寒门晋升"在 D2 中有明确支持
→ 输出最终回答

💻 实现代码

class SelfRAG:
    """自我反思的检索增强生成"""

    def __init__(self, llm, retriever, max_retrievals=3):
        self.llm = llm
        self.retriever = retriever
        self.max_retrievals = max_retrievals

    def should_retrieve(self, question: str) -> bool:
        """步骤 1：判断是否需要检索"""
        prompt = f"""判断以下问题是否需要从知识库中检索信息才能准确回答。
如果需要检索，输出 [Retrieve]；如果不需要，输出 [No Retrieve]。

问题：{question}

判断："""
        response = self.llm.invoke(prompt).content.strip()
        return "[Retrieve]" in response

    def is_relevant(self, question: str, document: str) -> bool:
        """步骤 2：判断检索结果是否相关"""
        prompt = f"""判断以下文档是否与问题相关。
如果相关，输出 [Is Relevant]；如果不相关，输出 [Not Relevant]。

问题：{question}
文档：{document}

判断："""
        response = self.llm.invoke(prompt).content.strip()
        return "[Is Relevant]" in response

    def is_supported(self, answer: str, documents: list[str]) -> bool:
        """步骤 3：判断回答是否有文档支持"""
        context = "\n".join(f"文档{i+1}：{d}" for i, d in enumerate(documents))
        prompt = f"""判断以下回答是否有文档中的事实支持。
如果有支持，输出 [Is Supported]；如果无支持，输出 [Not Supported]。

{context}

回答：{answer}

判断："""
        response = self.llm.invoke(prompt).content.strip()
        return "[Is Supported]" in response

    def generate(self, question: str, documents: list[str]) -> str:
        """基于文档生成回答"""
        if not documents:
            return self.llm.invoke(question).content
        context = "\n".join(f"文档{i+1}：{d}" for i, d in enumerate(documents))
        prompt = f"""根据以下文档回答问题。如果文档不足以回答，请说明。

{context}

问题：{question}
回答："""
        return self.llm.invoke(prompt).content

    def ask(self, question: str) -> dict:
        """完整的 Self-RAG 流程"""
        result = {"question": question, "retrieved": [], "answer": ""}

        # 步骤 1：是否需要检索？
        if not self.should_retrieve(question):
            result["answer"] = self.llm.invoke(question).content
            result["skipped_retrieval"] = True
            return result

        # 步骤 2：检索 + 相关性过滤
        documents = []
        for _ in range(self.max_retrievals):
            docs = self.retriever.invoke(question)
            for doc in docs:
                content = doc.page_content
                if self.is_relevant(question, content):
                    documents.append(content)
            if documents:
                break  # 找到相关文档就停止

        result["retrieved"] = documents

        # 步骤 3：生成回答
        answer = self.generate(question, documents)
        result["answer"] = answer

        # 步骤 4：支持度检查
        if documents and not self.is_supported(answer, documents):
            result["answer"] = "抱歉，知识库中没有足够信息回答这个问题。"
            result["unsupported"] = True

        return result


# 使用
self_rag = SelfRAG(llm, retriever)
result = self_rag.ask("谢初的官职是什么？")
print(f"回答：{result['answer']}")
print(f"检索了 {len(result['retrieved'])} 篇文档")

🎯 适用场景

高精度要求 — 医疗、法律、金融等领域，幻觉代价高，Self-RAG 的支持度检查能有效降低风险
混合查询 — 用户问题有的需要检索、有的不需要，Self-RAG 能自适应判断
知识库质量不稳定 — 检索结果可能包含无关文档，Self-RAG 的相关性过滤能自动排除噪音
不适合的场景 — 对延迟要求极高（如实时聊天），Self-RAG 的多轮反思会增加 2-4 倍的 LLM 调用

⚖️ 代价：延迟与成本增加

Self-RAG 不是免费的午餐。每次问答可能需要多次 LLM 调用：

调用步骤普通 RAGSelf-RAG
检索决策0 次1 次
相关性判断0 次1-N 次（每篇文档一次）
生成回答1 次1 次
支持度检查0 次1 次
总计1 次3-5 次
优化策略 1：相关性判断可以用轻量级分类器替代 LLM，减少延迟和成本
优化策略 2：支持度检查可以只对"不确定"的回答执行，先用一个置信度阈值过滤
优化策略 3：批量处理相关性判断——把所有文档拼在一起，一次 LLM 调用判断所有文档的相关性
