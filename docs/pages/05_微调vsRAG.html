微调 vs RAG

两种增强 LLM 能力的方案，如何选择？

🎯 核心区别

微调（Fine-tuning）和 RAG 都是让 LLM 获得额外知识/能力的方式，但机制完全不同。

维度微调（Fine-tuning）RAG
知识存储模型权重内部（参数化记忆）外部知识库（非参数化）
知识更新需要重新训练，成本高更新知识库即可，成本低
答案可追溯❌ 无法知道答案来源✅ 可追溯到具体文档
成本高（GPU 训练，数据标注）低（检索 + API 调用）
幻觉风险可能（模型可能记住错误知识）较低（有文档依据）
适用场景改变模型行为/风格/格式提供领域知识/实时信息

🧑‍💼 形象比喻

微调 = 培训员工

你花几个月培训一个员工
让他掌握专业知识
↓
他记住了这些知识
但无法告诉你"这是从哪学来的"
↓
知识更新需要重新培训
耗时耗力

RAG = 百科全书

你给员工一本百科全书
他不需要记住所有内容
↓
遇到问题时查阅相关章节
可以告诉你"这是第几页写的"
↓
知识更新只需换本书
简单快捷

🔍 什么时候用微调？

需要改变输出风格 — 让模型用特定语气、格式或风格回复（如客服话术、品牌调性）
需要遵循特定格式 — 生成固定结构的 JSON、SQL、代码等
需要学习特定推理模式 — 如法律推理、医学诊断逻辑
离线环境 — 无法联网，不能依赖外部知识库

🔍 什么时候用 RAG？

知识需要频繁更新 — 如新闻、产品信息、政策法规
需要答案可追溯 — 如法律咨询、医疗建议、学术研究
领域知识丰富 — 如企业内部文档、技术手册、小说知识库
预算有限 — 不想投入大量 GPU 训练成本

🔧 LoRA 轻量级微调原理

全量微调需要更新模型所有参数（如 70 亿个），成本极高。LoRA（Low-Rank Adaptation）通过低秩矩阵分解大幅减少训练参数。

LoRA 原理

原始权重矩阵 W（1024×1024 = 1,048,576 参数）
↓ 冻结不动
新增两个小矩阵：A（1024×8）+ B（8×1024）
只需训练 16,384 参数，减少 98.4%
↓
推理时：W' = W + A × B
效果接近全量微调，成本大幅降低
# LoRA 微调示例（使用 Hugging Face PEFT）
from peft import LoraConfig, get_peft_model

lora_config = LoraConfig(
    r=8,                # 秩（rank），越小参数越少
    lora_alpha=32,      # 缩放因子
    target_modules=["q_proj", "v_proj"],  # 只微调注意力层的 Q 和 V
    lora_dropout=0.05,
    bias="none"
)

model = get_peft_model(base_model, lora_config)
# 可训练参数从 7B → 约 4M（减少 99.9%）
LoRA 优势：训练成本极低（单卡 GPU 即可）、可切换不同 LoRA 适配器应对不同任务、不改变原始模型。
适用场景：数据量不大（几百到几千条）、需要快速迭代、资源有限。
