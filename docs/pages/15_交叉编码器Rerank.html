交叉编码器 Rerank

用交叉编码器精排检索结果，提升最终答案质量

🎯 什么是 Rerank？

Rerank 是对检索结果的二次排序。

普通检索（双塔模型）速度快但精度有限。Rerank 用更精准的模型对候选文档重新排序，选出最相关的 Top K。

🏗️ 双塔 vs 交叉编码器

维度双塔（Embedding）交叉编码器（Rerank）
输入问题和文档分别编码问题和文档一起输入
交互独立编码，算余弦相似度Cross-Attention 深度交互
速度快（可预计算）慢（每次都要算）
精度一般高
用途大规模初筛（100万→20）小规模精排（20→3）

→

📊 LLM Rerank vs 交叉编码器 Rerank

维度LLM Rerank交叉编码器 Rerank
模型GPT/Claude 等大模型专用小模型（bge-reranker）
速度慢（N 个文档 = N 次 API）快（1 次推理）
精度中高
成本高（按 token 计费）低（本地运行）

💻 实现代码

from sentence_transformers import CrossEncoder

# 加载交叉编码器模型
reranker = CrossEncoder('BAAI/bge-reranker-v2-m3')

# 组成 (question, doc) 对
pairs = [
    ("沈令月的驸马是谁？", "沈令月选谢初为驸马"),
    ("沈令月的驸马是谁？", "今天天气很好"),
    ("沈令月的驸马是谁？", "谢初是昭武将军"),
]

# 一次性推理，得到所有分数
scores = reranker.predict(pairs)
# scores = [0.95, 0.02, 0.78]

# 按分数排序
ranked = sorted(zip(pairs, scores), key=lambda x: x[1], reverse=True)

📦 常见 Rerank 模型

模型语言特点
BAAI/bge-reranker-v2-m3多语言中文效果好，开源（推荐）
BAAI/bge-reranker-large多语言更大，更准，更慢
Cohere Rerank多语言商业 API，效果好

💡 实际经验

粗筛用双塔，精排用交叉编码器 — 先用 Embedding 快速筛出 20 个候选，再用交叉编码器精排选出 3 个
候选数量很重要 — 粗筛太少（< 10）可能漏掉相关文档，太多（> 50）会增加 Rerank 时间
降级方案 — 如果无法下载交叉编码器模型，可以用 LLM 打分作为替代
