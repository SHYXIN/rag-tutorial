# 微调 vs RAG

## 核心区别

| | 微调（Fine-tuning） | RAG（检索增强生成） |
|--|---------------------|-------------------|
| **知识存储** | 模型权重内部 | 外部知识库 |
| **知识更新** | 需要重新训练 | 更新知识库即可 |
| **答案可追溯** | ❌ 无法追溯 | ✅ 可追溯到具体文档 |
| **成本** | 高（GPU 训练） | 低（检索 + API 调用） |
| **幻觉风险** | 可能（模型可能记错） | 较低（有文档依据） |
| **适用场景** | 行为/风格优化 | 知识库问答、文档检索 |

## 形象比喻

**微调** = 培训一个员工，让他记住所有知识
- 培训成本高（需要大量时间和资源）
- 知识更新难（需要重新培训）
- 但行为一致性好（知道该怎么做）

**RAG** = 给员工一本百科全书，让他随时查阅
- 培训成本低（不需要重新培训）
- 知识更新容易（更新百科全书即可）
- 答案有据可查（可以追溯到具体页码）

## 什么时候用微调？

1. **改变模型行为** — 让模型用特定语气、风格回答
2. **特定任务优化** — 代码生成、医疗诊断、法律分析
3. **知识已经稳定** — 知识不需要频繁更新
4. **需要低延迟** — 不需要检索步骤，直接生成

## 什么时候用 RAG？

1. **知识库问答** — 企业内部文档、FAQ
2. **知识频繁更新** — 新闻、产品信息、政策法规
3. **需要可追溯** — 医疗、法律、金融等需要引用的场景
4. **多数据源** — 需要从多个文档中综合信息

## 两者可以结合

```
微调 + RAG = 最佳实践

1. 先用微调优化模型的基础能力（语气、格式、专业术语）
2. 再用 RAG 提供实时知识
3. 效果：既专业又准确，还能追溯来源
```

## LoRA：轻量级微调

全量微调成本太高（需要更新所有参数），**LoRA** 是一种轻量级微调方法：

```
原始模型参数：W（不变）
LoRA 新增：A × B（只训练 A 和 B）
最终输出：W × x + A × B × x

其中 A 和 B 是低秩矩阵，参数量远小于 W
```

**LoRA 的优势：**
- 训练成本低（只需要训练少量参数）
- 可以切换不同任务（换不同的 A、B 矩阵）
- 不改变原始模型（随时可以恢复）
